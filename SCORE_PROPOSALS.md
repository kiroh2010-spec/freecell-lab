# 점수 개편안

## 1안 - 균형형 + 3분 미만 보너스

상태: 토론용 저장안. 아직 dev/alpha/beta 미적용.

```txt
기본점수 = 3000
시간점수 = max(0, 900 - 클리어초 × 2)
3분미만보너스 = 클리어초 < 180 이면 +100, 아니면 0
이동점수 = max(0, 900 - 이동수 × 7)
되돌리기패널티 = 되돌리기사용수 × 40

최종점수 =
(기본점수 + 시간점수 + 3분미만보너스 + 이동점수 - 되돌리기패널티)
× 난이도배수
```

### 의도

- 기존 9000점대 점수를 3000~4000점대로 낮춘다.
- 3분 미만 속도형 플레이에 확실한 가산점을 준다.
- 이동 수가 적은 계산형 플레이도 경쟁 가능하게 한다.
- 100수 전후 일반 플레이를 과하게 죽이지 않는다.
- 되돌리기 패널티는 유지하되 과하지 않게 한다.

### 현재 랭커 기준 관찰

- 2분대/90~100수 플레이가 상위권을 유지한다.
- 3분대/80수대 플레이도 1위권 경쟁이 가능하다.
- 5분대/70수대 플레이는 보상은 받지만, 시간 차이 때문에 상위권 진입은 다소 어렵다.

## 2안 - 프리셀 퍼즐형

상태: 토론용 저장안. 아직 dev/alpha/beta 미적용.

```txt
클리어점수 = 2000
이동보너스 = max(0, 1000 - 이동수 × 7)
시간보너스 = max(0, 600 - 클리어초 × 1.5)
되돌리기패널티 = 되돌리기사용수 × 40

최종점수 =
(클리어점수 + 이동보너스 + 시간보너스 - 되돌리기패널티)
× 난이도배수
```

### 의도

- 프리셀을 “빨리 깨는 게임”보다 “깔끔하게 풀고, 빠르면 더 좋은 퍼즐 게임”으로 본다.
- 이동 수 보상을 시간 보상보다 크게 둔다.
- 계산형/균형형 플레이가 랭킹에서 살아남게 한다.

## 최종 후보 - 단일 공식으로 레벨별 체감 유도

상태: dev 가상 개편 랭킹에 반영. 서버/알파/베타에는 아직 미적용.

레벨별로 공식을 바꾸자는 뜻이 아니라, **하나의 공식**을 쓰되 레벨 난이도 차이 때문에 경쟁 성격이 자연스럽게 달라지는지 검토한다.

목표 체감:

```txt
1LV 경쟁 = 타임어택 성향
2LV 경쟁 = 시간 + 퍼즐 균형
3LV 경쟁 = 퍼즐 성향
```

현재 dev 개편 랭킹 공식:

```txt
클리어점수 = 2000

시간보너스 원점수 =
  5분까지: max(0, 1000 - 클리어초 × 3)
  5분~10분: max(0, 100 - (클리어초 - 300) / 3)
시간보너스 = 시간보너스 원점수 × 1.5

이동보너스 원점수 =
  120무브까지: max(0, 900 - 이동수 × 7)
  120~200무브: max(0, 60 - (이동수 - 120) × 0.75)
이동보너스 = 이동보너스 원점수 × 1.5

되돌리기패널티 = 되돌리기사용수 × 40
필살기패널티 = 필살기사용시 200

최종점수 =
(클리어점수 + 시간보너스 + 이동보너스 - 되돌리기패널티 - 필살기패널티)
× 레벨배수
```

### 왜 단일 공식으로도 레벨별 체감이 바뀔 수 있나

- 1LV는 딜이 쉬워 시간이 짧고 이동 수 범위가 비교적 좁다. 그래서 시간보너스 차이가 크게 느껴져 타임어택 성향이 살아난다.
- 2LV는 시간이 조금 길어지고 이동 수 차이도 커지므로 시간/이동이 같이 중요해진다.
- 3LV는 시간이 5분 33초를 넘기면 시간보너스가 0에 가까워지고, 이동보너스가 핵심 경쟁축이 된다. 그래서 퍼즐형 체감이 자연스럽게 강해진다.

주의점:

- 시간보너스가 5분 33초 이후 0이 되므로, 3LV에서 6분과 10분의 시간 차이는 점수에 직접 반영되지 않을 수 있다.
- 이것을 의도적인 “고레벨 퍼즐화”로 볼지, 너무 극단적인 시간 무시로 볼지는 추가 테스트가 필요하다.
