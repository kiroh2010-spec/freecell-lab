# Freecell 난이도 / 레벨업 / 점수 규칙

## 방향

- 난이도는 3단계만 사용한다: **1LV → 2LV → 3LV**.
- 내부 코드는 호환성을 위해 `e1/e2/n1`을 유지한다.
- 기존 `n2/n3` 저장값은 새 정책에서 `n1(3LV)`로 흡수한다.
- 사용자 화면에서는 Easy/Normal/Hard 같은 표현을 쓰지 않는다.
- 낮은 단계도 “쉬운 모드”가 아니라, 부담 없이 시작하는 좋은 선택지로 보이게 한다.

## 난이도 단계

| 표시명 | 한국어 | 내부 코드 | 필요 누적 클리어 | 배수 | 의도 |
| --- | --- | --- | ---: | ---: | --- |
| 1LV | 기본 | e1 | 0 | 1.00x | 부담 없이 한 판 시작하는 기본 단계 |
| 2LV | 표준 | e2 | 3 | 1.08x | 랭킹을 노리는 표준 단계 |
| 3LV | 승부 | n1 | 6 | 1.20x | 집중해서 붙는 승부 단계, 필살기 해금 |

## 레벨업 테스트

- 현재 난이도에서 다음 단계의 필요 누적 클리어 수를 만족하면, 다음 새 게임은 **레벨업 테스트**로 시작한다.
- 레벨업 테스트는 **현재 단계 난이도의 딜**로 진행한다. 예: 1LV → 2LV 레벨업 테스트는 1LV 딜, 2LV → 3LV 레벨업 테스트는 2LV 딜.
- 레벨업 테스트는 제한 시간 **7분** 안에 클리어하면 성공한다.
- 레벨업 테스트 점수는 목표 난이도 배수에 **+0.10x 보너스**를 더한다.
- 레벨업 테스트 클리어 시 해당 단계로 레벨업한다.

예시:

- 1LV에서 누적 3클리어 달성
- 다음 새 게임은 2LV 레벨업 테스트
- 클리어하면 2LV로 레벨업

## 점수 규칙

```txt
기본점수 = 10000 - 이동수 × 5 - 힌트사용수 × 100 + 시간보너스
최종점수 = 기본점수 × 난이도/레벨업 테스트 배수
```

시간은 연속 감점이 아니라 컷오프 보너스다. 오래 걸려도 추가 시간 감점은 없다.

| 클리어 시간 | 보너스 |
| --- | ---: |
| 3분 이내 | +200 |
| 4분 이내 | +100 |
| 5분 이내 | +50 |
| 5분 초과 | +0 |

## 난이도 판단 기준

현재는 완전 solver가 아니라, 딜 결과를 평가해 난이도에 맞지 않으면 다시 셔플하는 휴리스틱 방식을 사용한다.

- 1LV: 현재 테스트 기준 유지. 초반 저카드/이동 가능 카드가 넉넉한 판. `total <= 6`, 맨 위 A/2/3 최소 3장, 시작 이동 가능 카드 최소 4장 기준.
- 2LV: 1LV보다 시작 선택지가 살짝 줄어드는 표준 판. `total 3~5`, 맨 위 A/2/3 최소 3장, 시작 이동 가능 카드 최소 3장 기준.
- 3LV: 2LV보다 A 접근성과 초반 선택지가 조금 더 빡빡한 승부용 판. `total 5~7`, 맨 위 A/2/3 최소 3장, 시작 이동 가능 카드 최소 3장 기준.
