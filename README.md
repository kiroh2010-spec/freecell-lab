# Freecell Project

현재 집중할 메인 프로젝트입니다.

## 현재 DEV

브라우저에서 바로 플레이 가능한 단일 페이지 프리셀 개발판입니다.

현재 DEV 버전: v0.9

- 진입 파일: `index.html`
- 스타일: `style.css`
- 게임 로직: `script.js`


## 버전 관리 규칙

용어는 `dev / 알파 / 베타 / 런칭`으로 고정한다. 기본 흐름은 **dev에서 개발 → 알파에서 사람 테스트 → 검증된 알파를 베타에 거의 그대로 적용 → 런칭**이다.

- **dev**
  - 로컬 개발/테스트 공간이다.
  - 테스트 기능, 테스트 데이터, 치트성 도구를 그대로 유지한다.
  - 화면 표시는 `DEV v0.x` 계열을 사용한다.
  - 버전은 포함 스펙 규모에 따라 증가한다. 메이저급 변경은 `+0.1`, 마이너/트레비얼 변경은 `+0.01`.
  - 기준 필드: `VERSION.json`의 `devVersion`.
- **알파**
  - dev 변경을 먼저 공개 테스트하는 공간이다.
  - 링크를 통해 사람 테스트를 진행하되, 베타 사용자에게 바로 영향을 주지 않는다.
  - 테스트 관련 기능/데이터는 UI와 코드에서 제거한다.
  - 새 실험 흐름은 `알파 v0.1`부터 시작한다.
  - 기준 필드: `VERSION.json`의 `alphaPublicVersion`.
  - 공개 경로: `/alpha/`.
- **베타**
  - 알파에서 사람 테스트를 통과한 내용을 적용하는 안정 테스트 공간이다.
  - 알파에서 베타로 적용할 때는 큰 재작업 없이 **거의 그대로 반영**하는 것을 목표로 한다.
  - 기존 공개 사용자가 보는 기본 링크는 베타로 유지한다.
  - 버전은 `베타 v0.1`부터 시작하고 이후 포함 스펙 규모에 따라 증가한다.
  - 기준 필드: `VERSION.json`의 `betaVersion`.
  - 공개 경로: `/`.
- **런칭**
  - 팀장이 “런칭하자”라고 하면 이 채널로 간다.
  - 치트/테스트 관련 내용은 절대 포함하지 않는다.
  - `1.0`부터 시작하고 이후 스펙 규모에 따라 관리한다.
  - 기준 필드: `VERSION.json`의 `launchVersion`.

`scripts/build-pages.sh`는 공개 빌드를 생성한다. 베타는 `docs/index.html`, `docs/script.js`, `docs/VERSION.json`에 반영되고, 알파는 `docs/alpha/index.html`, `docs/alpha/script.js`, `docs/alpha/VERSION.json`에 반영된다. 자산 캐시는 각각 `beta-0-1`, `alpha-0-1`처럼 채널 버전으로 관리한다.

## 플레이 방법

1. `index.html`을 브라우저로 엽니다.
2. 이동할 카드를 클릭한 뒤 목적지를 클릭합니다.
3. 또는 이동 가능한 카드를 드래그해서 Free Cell, Foundation, Tableau 열에 놓습니다.
4. Tableau의 맨 아래 카드나 Free Cell 카드를 더블클릭하면, 가능한 경우 Foundation으로 바로 이동합니다.
5. Tableau 맨 아래 카드가 Foundation으로 갈 수 없으면, 더블클릭 시 빈 Free Cell로 이동합니다.
6. `되돌리기`는 한 게임당 5번까지 사용할 수 있고, 직전 이동을 되돌립니다.
6. `소리 켜짐/꺼짐` 버튼으로 이동 사운드를 제어합니다.

## 현재 구현된 룰

- 52장 셔플 후 8개 Tableau 열에 배치
- 각 Tableau 열의 맨 아래 카드만 이동 가능
- Free Cell 4칸 사용 가능
- Foundation은 A부터 같은 무늬로 오름차순
- Tableau는 다른 색상 + 내림차순으로 이동
- Tableau에서 색이 번갈아 내려가는 연속 숫자 묶음은 한 번에 이동 가능
- 빈 Tableau 열에는 아무 카드나 또는 유효한 연속 묶음 이동 가능
- 새 게임 버튼
- Foundation으로 가능한 카드 자동 이동 버튼
- 데스크톱 브라우저 드래그 앤 드롭
- Tableau 맨 아래 카드 또는 Free Cell 카드 더블클릭 시 Foundation 이동
- Tableau 맨 아래 카드 더블클릭 시 Foundation 불가하면 빈 Free Cell로 자동 이동
- 겹친 Tableau에서도 숫자+마크가 같이 보이는 카드 상단 표시
- 이동 성공 / Foundation 이동 / 불가능한 이동 / 승리 사운드
- 소리 켜기/끄기 버튼
- Foundation에 카드가 쌓이는 레이어 표현
- 한 게임당 5회 제한 되돌리기
- 모바일 화면에서 하단 고정 메뉴 배치
- 모바일 한 화면에 최대한 들어오도록 상단/보드/카드 간격 압축
- 모바일 Tableau 영역이 과하게 길어지지 않도록 빈 테이블 공간 비율 조정
- J/Q/K 얼굴카드는 A와 구분되도록 전용 배지/마크 표시
- iPhone 카메라/노치 영역을 고려한 상단 safe-area 여백 적용
- 새로고침/탭 재방문 시 진행 중인 게임 자동 저장/복원
- 새 게임 버튼을 누를 때만 현재 판 초기화
- 모바일 상단 정보 밀도를 낮추기 위해 타이틀/플레이어와 랭킹 영역을 2줄로 분리
- 버전 표시는 `DEV v0.9`로 단순화하고, 플레이어 카드에는 현재 난이도명(`1LV/2LV/3LV`)을 표시
- HUD 순서를 MOVE / TIME / UNDO로 조정하고 TIME을 중앙 강조
- 최초 접속자는 1LV에서 시작
- 클리어 횟수에 따라 1LV → 2LV → 3LV까지 레벨업
- 난이도 상승 시 피버 대신 레벨업 테스트 발동
- 레벨업 테스트는 현재 단계 난이도를 7분 안에 클리어하면 다음 난이도로 레벨업하고 랭킹에도 기록
- 현재 최대 난이도는 3LV이며, 더 높은 단계는 유저 반응을 보고 추후 업데이트
- 높은 난이도일수록 점수 배수가 높아져 상위 난이도 유저가 높은 점수를 얻기 쉬운 밸런스
- 점수 계산에 되돌리기 사용 횟수 반영: 되돌리기 1회당 -100점
- 소리 버튼을 텍스트/이모지 대신 검은색 간소화 스피커 심볼 UX로 개선
- PC 와이드 화면에서도 게임 영역이 과하게 늘어나지 않도록 최대폭/중앙 정렬 적용
- 게임 타이머
- 새 게임 첫 화면에서 Foundation으로 바로 보낼 수 있는 A 카드 1장 보장
- 첫 접속 시 플레이어 ID와 숫자 4자리 PW 자동 생성
- PLAYER 영역 클릭 시 ID/PW 수정 가능하되 최초 1회만 허용
- 수정 패널에서 ID/PW 변경은 최초 1회만 가능하다고 안내
- ID 변경 시 기존 로컬 랭킹 기록의 ID도 새 ID로 반영
- 같은 ID/숫자 4자리 PW를 입력하면 로컬에 저장된 기존 클리어/난이도 정보를 계승
- 등록된 플레이어는 화면에 ID와 PW 보기 토글 표시
- 카드 선택 시 Tableau 안에서 이동 가능한 열을 눈에 띄게 표시
- 카드 선택 상태에서 보드 바깥 빈 영역 클릭 시 선택 표시 해제
- 주간 랭킹 상단은 1위 고정 + 2~5위 2초 순환 티커로 표시
- 플레이어 카드에는 내 랭킹, 금/은/동 트로피, 현재 난이도명 표시
- 주간 랭킹 클릭 시 TOP 50 상세 순위, 점수, 등록 시간 표시
- 클리어 시 결과창 표시: 시간/무브/점수 중앙 표시, 확인 시 주간 랭킹 반영 연출
- 주간 랭킹 미진입 시 TOP20까지 부족한 점수 안내
- 랭킹 목록의 중복 순번 표시 제거
- 주간 랭킹을 상단 중앙 영역에 compact 표시하고 다음 초기화 예정일 표시
- Supabase 서버 랭킹 연동 준비: 플레이어 등록/수정, 점수 제출, 주간 리더보드 조회 RPC 연결
- 플레이 중 30초마다 서버 랭킹을 갱신하고 1위 변경/1위와 점수 차이 알림
- 랭킹 기록이 없을 때 빈 문구를 표시하지 않음
